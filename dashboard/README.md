# Bailey Partnership Software Survey Dashboard

Interactive web dashboard for the IIET Committee Software Survey Analysis.

## Overview

This dashboard presents comprehensive analysis of software usage patterns across Bailey Partnership Group, including:

- Response demographics (discipline, role level, office location)
- Software usage analysis with satisfaction metrics
- Training needs assessment
- Software churn and demand analysis
- Custom software discovery
- Discipline-specific insights
- Strategic recommendations

## Features

- **Responsive Design**: Works on desktop, tablet, and mobile devices
- **Interactive Charts**: Powered by Chart.js for dynamic visualizations
- **Sortable Tables**: Click column headers to sort data
- **Print-Friendly**: Optimized stylesheet for printing reports
- **Smooth Navigation**: Fixed sidebar with section links

## Deployment to GitHub Pages

### Option 1: Using GitHub Web Interface

1. Create a new GitHub repository
2. Upload all files maintaining this folder structure:
   ```
   /
   ├── index.html
   ├── README.md
   ├── css/
   │   └── styles.css
   ├── js/
   │   ├── app.js
   │   ├── charts.js
   │   ├── data.js
   │   └── navigation.js
   └── assets/
   ```
3. Go to **Settings** > **Pages**
4. Under "Source", select **Deploy from a branch**
5. Choose **main** branch and **/ (root)** folder
6. Click **Save**
7. Dashboard will be available at: `https://[username].github.io/[repo-name]/`

### Option 2: Using Git Command Line

```bash
# Clone or initialize repository
git init bailey-survey-dashboard
cd bailey-survey-dashboard

# Copy all dashboard files to this directory

# Add and commit
git add .
git commit -m "Initial dashboard deployment"

# Push to GitHub
git remote add origin https://github.com/[username]/[repo-name].git
git push -u origin main

# Enable GitHub Pages via repository settings
```

## Local Preview


```bash

# Follow the URL shown in terminal
```

### Direct File Opening
Simply open `index.html` in a web browser. Note: Some features may be limited due to CORS restrictions.

## Updating Data

To update the dashboard with new survey responses:

1. Export new CSV from Google Sheets
2. Run the Python parsing script to generate new analysis:
   ```bash
   python parse_survey.py > analysis.json
   ```
3. Update `js/data.js` with the new data
4. Commit and push changes

## Customization

### Adding New Sections

1. Add section HTML to `index.html`
2. Add navigation link to sidebar
3. Create data population function in `app.js`
4. Add any charts in `charts.js`

### Modifying Charts

Edit `js/charts.js` to change:
- Chart types
- Color schemes
- Axis configurations
- Tooltip formats

### Styling Changes

Edit `css/styles.css` to modify:
- Colors (via CSS custom properties)
- Typography
- Layout
- Component styles
